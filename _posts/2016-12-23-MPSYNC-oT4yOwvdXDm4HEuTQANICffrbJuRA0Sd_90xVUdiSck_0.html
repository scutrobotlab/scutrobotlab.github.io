Opencv简介

OpenCV的全称是：Open Source Computer Vision Library。OpenCV是一个基于BSD许可（开源）发行的跨平台计算机视觉库，可以运行在Linux、Windows和Mac OS操作系统上。

它轻量级而且高效——由一系列 C 函数和少量 C++ 类构成，同时提供了Python、Ruby、MATLAB等语言的接口，实现了图像处理和计算机视觉方面的很多通用算法。

OpenCV于1999年由Intel建立，如今由Willow Garage提供支持。目前最新版本是3.1，2016年1月29日发布。


opencv为我们提供了五百多个C\C++函数，这些跨平台的函数具有强大的图像和矩阵处理的运算能力，能够帮助我们解决实际的工程问题。

但是，要想把这所有的函数全都掌握是不可能的

所以，我们首先得对它有一个全局的把握，了解它的各部分模块，

这样才能更好的找到我们需要的函数并加以利用

 


opencv2.4.10包含的模块和主要功能


图文解释，一大波看不懂的英文名要来啦~

  

calib3d：相机校准与三维重建模块，包括：摄像机标定、立体匹配、3D   重建等。

contrib：新贡献的模块，包含一些开发者新贡献出来的尚不成熟的代码。

core：核心功能模块，包括基本的数据结构、矩阵运算、傅里叶变换、文件IO等。

features2d：二维特征检测与描述模块，包括：图像特征检测、描述、匹配等。

flann：即FLANN（fast library for approximate nearest neighbors）接口模块、FLANN库中包含在高维空间中进行搜索和聚类的方法。

gpu：GPU加速模块，包括一些可以利用cuda进行加速的函数。

highgui：高层用户交互模块，包括图形界面、图像与视频IO等。

imgproc：图像处理模块，包括图像滤波、形态学处理、几何变换、颜色  空间转换及直方图计算等。

legacy：遗留模块，包括一些过期的代码，用于保持前后兼容。

ml：机器学习模块，包含一些主流的机器学习算法，如支持向量机、决策树、boosting。

nonfree：一些具有专利的算法模块，包含特征检测（SIFT）和GPU相关的内容，不可商用。

objdetect：目标检测模块，包括基于Haar特征或HBP特征的人脸检测、基于HOG特征的人体检测等。

ocl：即OpenCL-accelerated Computer Vision，运用OpenCL加速的计算机视觉组件模块。

photo：即Computational Photography，包含图像修复和图像去噪两部分。

stitching：图像拼接模块，包含图像拼接、匹配、自动校准、曝光补偿等部分。

superres：SuperResolution，超分辨率技术的相关功能模块。

ts：opencv测试的相关代码，不用管。

video：视频分析模块，包含运动预测、目标跟踪、背景减除、光流等。

videostab：Video stabilization，视频稳定相关的组件。

opencv的core内核为我们提供了许多方便易用的类和接口，就拿最基本的cv::Mat类型来说吧，它将我们图像的所有像素点数据以矩阵的形式存储，更适合计算机做数值运算，也方便我们进行各类矩阵操作。

比方说我们要做像素遍历，有很多方法：


1.利用Mat类的成员函数at(int y,int x)逐点访问:

    image.at<uchar>(j,i) = 255;


2.利用与STL迭代器兼容的cv::Mat迭代器:

cv::Mat_<cv::Vec3b>::iterator it = image.begin<cv::Vec3b>();


3.利用指针进行内存访问:

uchar *data = image.data;


上述三种方法中，第1种方法的效率最低，

如果不是需要随机访问像素点的话，不建议使用。


下图就是利用指针操作方法（第3种）来实现全图像素遍历，实现图像颜色缩减和空间转换，这一处理有利于去除无用信息和噪声，放大颜色间的差异，有利于后期的特征分析和提取。
